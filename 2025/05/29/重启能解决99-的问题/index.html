重启能解决99%的问题

前言

在之前对springboot项目进行打包部署时遇到了maven版本过低的问题，于是在今天下午升级了maven版本
，然后测试完打包部署后进行对themeleaf基础的一些实现。于是问题就出现了。

实现th：object时的报错

mythymeleaf] [nio-8080-exec-7] o.a.c.c.C.[.[.[/].[dispatcherServlet] : Servlet.service() for servlet [dispatcherServlet] in context with path [] threw exception [Request processing failed: org.thymeleaf.exceptions.TemplateProcessingException: Exception evaluating SpringEL expression: “sname” (template: “object” - line 9, col 12)] with root cause

org.springframework.expression.spel.SpelEvaluationException: EL1008E: Property or field ‘sname’ cannot be found on object of type ‘com.example.mythymeleaf.entity.Supplier’ - maybe not public or not valid?

分析

大概意思是找不到sname的实现，但是在检查完代码和依赖没问题后仍然无法解决
于是便继续进行下一个案例的实现

实现表单提交th：field时的报错

java: 找不到符号
符号: 方法 setSname(java.lang.String)
位置: 类型为com.example.mythymeleaf.entity.Supplier的变量 supplier

分析

在检查完lombok的依赖，supplier的代码还有控制器的代码实现以及html的代码后仍无法解决，而直接关闭再打开IDEA，maven clean and install也没有效果。
最后，在上网搜索后发现了解决方法


然后发现报错1也解决了

两种重启方式对比

操作方式 清理缓存 速度 适用场景
直接叉掉 ❌ 快 日常关闭
Invalidate and Restart ✅ 全部 慢 解决顽固性缓存问题
